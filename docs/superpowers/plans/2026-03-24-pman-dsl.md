# .pman DSL Parser Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement a parser for the new `.pman` DSL format so procman can load `.pman` files alongside the existing YAML format.

**Architecture:** Hand-written lexer and recursive-descent parser. The lexer tokenizes `.pman` source into a stream of typed tokens. The parser consumes tokens into an AST. A validation pass checks structural rules. A lowering pass converts the AST into the existing `ProcessConfig` types that the runtime already understands. The `.pman` and `.yaml` codepaths share all runtime code — only the parsing frontend differs.

**Tech Stack:** Rust (edition 2024), no new dependencies. Uses existing `anyhow`, `glob`, `serde_json_path` crates.

**Spec:** `docs/superpowers/specs/2026-03-24-pman-dsl-design.md`

```rust
/// Source location for error reporting.
#[derive(Clone, Copy, Debug, PartialEq, Eq)]
pub struct Span {
    pub start: usize, // byte offset
    pub end: usize,   // byte offset
    pub line: usize,  // 1-based line number
    pub col: usize,   // 1-based column (byte offset within line)
}

impl Span {
    /// Format an error message with file location.
    pub fn fmt_error(&self, path: &str, msg: &str) -> String {
        format!("{path}:{}:{}: {msg}", self.line, self.col)
    }
}

#[derive(Clone, Debug, PartialEq)]
pub enum TokenKind {
    // Keywords
    Config,     // config
    Job,        // job
    Event,      // event
    If,         // if
    For,        // for
    In,         // in
    Env,        // env
    Run,        // run
    Wait,       // wait
    Watch,      // watch
    After,      // after
    Once,       // once
    OnFail,     // on_fail
    Spawn,      // spawn
    Http,       // http
    Connect,    // connect
    Exists,     // exists
    Contains,   // contains
    Running,    // running
    Glob,       // glob
    Arg,        // arg
    True,       // true
    False,      // false
    None,       // none

    // Literals
    String(String),       // "..." (contents, escapes resolved)
    Number(f64),          // 42, 3.14
    Duration(f64),        // 5.0 (seconds) — suffix parsed into seconds
    FencedString(String), // ``` ... ``` (raw contents)

    // Identifiers and references
    Ident(String),    // bare identifier (e.g., job name, var name)
    At,               // @
    Dot,              // .
    Args,             // args (keyword for args namespace)

    // Operators
    Eq,      // ==
    Ne,      // !=
    Lt,      // <
    Gt,      // >
    Le,      // <=
    Ge,      // >=
    And,     // &&
    Or,      // ||
    Not,     // !

    // Punctuation
    Assign,       // =
    LBrace,       // {
    RBrace,       // }
    LBracket,     // [
    RBracket,     // ]
    LParen,       // (
    RParen,       // )
    Comma,        // ,
    DotDot,       // ..
    DotDotEq,     // ..=
}

#[derive(Clone, Debug)]
pub struct Token {
    pub kind: TokenKind,
    pub span: Span,
}
```

- [ ] **Step 3: Implement the lexer**

Create `src/pman/lexer.rs` with a `pub fn lex(input: &str) -> Result<Vec<Token>>` function. The lexer maintains a running `line: usize` (starting at 1) and `col: usize` (starting at 1, reset on newlines) as it scans, stamping each token's `Span` with the current position. The lexer should:

- Track `line` and `col` — increment `line` and reset `col` to 1 on `\n`; increment `col` for each byte consumed otherwise
- Skip whitespace and `#` comments (updating line/col as it goes)
- Recognize all keywords (`config`, `job`, `event`, `if`, `for`, `in`, `env`, `run`, `wait`, `watch`, `after`, `once`, `on_fail`, `spawn`, `http`, `connect`, `exists`, `contains`, `running`, `glob`, `as`, `arg`, `true`, `false`, `none`, `args`)
- Parse double-quoted strings with `\"`, `\\`, `\n`, `\t` escapes
- Parse triple-backtick fenced strings (raw content between opening and closing `` ``` ``)
- Parse numbers (integer and float)
- Parse durations: number followed immediately (no whitespace) by `s`, `ms`, or `m` suffix (convert to seconds as f64). The suffix must be followed by a non-alphanumeric character or EOF — `5something` should lex as `Number(5.0)` then `Ident("something")`, while `5s` lexes as `Duration(5.0)`. Check for the longest matching suffix first (`ms` before `m`).
- Parse all operators and punctuation
- Parse identifiers: `[a-zA-Z_][a-zA-Z0-9_-]*`
- Track spans for error reporting
- `@` is its own token (not part of identifiers)
- `!` followed by `=` produces `Ne`; standalone `!` produces `Not`
- `..=` vs `..` — check for `=` after `..`

```rust
use crate::pman::token::Span;

/// A complete .pman file.
#[derive(Debug)]
pub struct File {
    pub config: Option<ConfigBlock>,
    pub jobs: Vec<JobDef>,
    pub events: Vec<EventDef>,
}

#[derive(Debug)]
pub struct ConfigBlock {
    pub logs: Option<StringLit>,
    pub env: Vec<EnvBinding>,
    pub args: Vec<ArgDef>,
    pub span: Span,
}

#[derive(Debug)]
pub struct ArgDef {
    pub name: String,
    pub arg_type: Option<ArgType>,
    pub default: Option<Expr>,
    pub short: Option<StringLit>,
    pub description: Option<StringLit>,
    pub span: Span,
}

#[derive(Debug, Clone, PartialEq)]
pub enum ArgType {
    String,
    Bool,
}

#[derive(Debug)]
pub struct JobDef {
    pub name: String,
    pub condition: Option<Expr>,
    pub body: JobBody,
    pub span: Span,
}

#[derive(Debug)]
pub struct EventDef {
    pub name: String,
    pub body: JobBody,
    pub span: Span,
}

#[derive(Debug)]
pub struct JobBody {
    pub once: Option<bool>,
    pub env: Vec<EnvBinding>,
    pub wait: Option<WaitBlock>,
    pub watches: Vec<WatchDef>,
    pub run_section: RunSection,
}

/// Either a direct run command or a for-loop wrapping env+run.
#[derive(Debug)]
pub enum RunSection {
    Direct(ShellBlock),
    ForLoop(ForLoop),
}

#[derive(Debug)]
pub struct ForLoop {
    pub var: String,
    pub iterable: Iterable,
    pub env: Vec<EnvBinding>,
    pub run: ShellBlock,
    pub span: Span,
}

#[derive(Debug)]
pub enum Iterable {
    Glob(StringLit),
    Array(Vec<Expr>),
    RangeExclusive(Expr, Expr),
    RangeInclusive(Expr, Expr),
}

#[derive(Debug)]
pub struct EnvBinding {
    pub key: String,
    pub value: Expr,
    pub span: Span,
}

#[derive(Debug)]
pub enum ShellBlock {
    Inline(StringLit),
    Fenced(String, Span),
}

#[derive(Debug)]
pub struct StringLit {
    pub value: String,
    pub span: Span,
}

/// Wait block — ordered list of conditions.
#[derive(Debug)]
pub struct WaitBlock {
    pub conditions: Vec<WaitCondition>,
    pub span: Span,
}

#[derive(Debug)]
pub struct WaitCondition {
    pub negated: bool,
    pub kind: ConditionKind,
    pub options: ConditionOptions,
    pub span: Span,
}

#[derive(Debug)]
pub enum ConditionKind {
    After { job: String },
    Http { url: StringLit },
    Connect { address: StringLit },
    Exists { path: StringLit },
    Running { pattern: StringLit },
    Contains { path: StringLit, format: String, key: StringLit, var: Option<String> },
}

#[derive(Debug, Default)]
pub struct ConditionOptions {
    pub status: Option<Expr>,
    pub timeout: Option<Expr>,
    pub poll: Option<Expr>,
    pub retry: Option<Expr>,
}

#[derive(Debug)]
pub struct WatchDef {
    pub name: String,
    pub condition: WaitCondition,
    pub initial_delay: Option<Expr>,
    pub poll: Option<Expr>,
    pub threshold: Option<Expr>,
    pub on_fail: Option<OnFailAction>,
    pub span: Span,
}

#[derive(Debug)]
pub enum OnFailAction {
    Shutdown,
    Debug,
    Log,
    Spawn(String),
}

/// Expression — evaluated at runtime.
#[derive(Debug, Clone)]
pub enum Expr {
    StringLit(String, Span),
    NumberLit(f64, Span),
    BoolLit(bool, Span),
    DurationLit(f64, Span),
    NoneLit(Span),
    ArgsRef(String, Span),         // args.name
    JobOutputRef(String, String, Span), // @job.KEY
    LocalVar(String, Span),        // bare identifier in expression context
    BinOp(Box<Expr>, BinOp, Box<Expr>, Span),
    UnaryNot(Box<Expr>, Span),
}

#[derive(Debug, Clone, Copy, PartialEq)]
pub enum BinOp {
    Eq, Ne, Lt, Gt, Le, Ge, And, Or,
}
```

- `pub fn lower(input: &str, extra_env: &HashMap<String, String>, arg_values: &HashMap<String, String>) -> Result<(Vec<ProcessConfig>, Option<String>)>`
  - Calls `parser::parse(input)` then `validate::validate(&file)`
  - Resolves global `config { env { } }` bindings eagerly (args are known) into a `global_env: HashMap<String, String>`
  - For each job/event: merge system env + extra_env + global_env + per-job env (respecting precedence)
  - Evaluate `if` conditions eagerly — skip jobs where condition is false (but register skipped `once = true` jobs so `after` dependents aren't blocked)
  - Note: `.pman` args have no `env` field (unlike YAML). Global env bindings replace that functionality. The `ArgDef`s returned from `parse_header` will have `env: None`; global env bindings are folded into jobs during lowering, not via the `main.rs` `merged_env` step.
  - Converts each `JobDef` and `EventDef` into a `ProcessConfig`
- Expression evaluation for env bindings:
  - `ArgsRef` → look up in `arg_values`, substitute eagerly
  - `JobOutputRef` → convert to `${{ job.KEY }}` string for runtime resolution
  - `LocalVar` → look up in a local var map (populated by `var` bindings from `contains`)
  - `StringLit`, `NumberLit`, `BoolLit` → convert to string
  - `DurationLit` → this is a type error in env context (durations are only for options)
- Condition lowering:
  - `After` → `Dependency::ProcessExited`
  - `Http` (not negated) → `Dependency::HttpHealthCheck`
  - `Connect` (not negated) → `Dependency::TcpConnect`
  - `Connect` (negated) → `Dependency::TcpNotListening`
  - `Exists` (not negated) → `Dependency::FileExists`
  - `Exists` (negated) → `Dependency::FileNotExists`
  - `Running` (negated) → `Dependency::ProcessNotRunning`
  - `Contains` → `Dependency::FileContainsKey`
- Watch lowering → `Watch` structs
- `for` loop → `ForEachConfig` (convert `Iterable` to the glob string for `glob()`, or expand arrays/ranges into multiple `ProcessConfig`s)
- `if` condition → evaluate eagerly at lower time. Since `if` conditions only reference `args.*` (which are known), evaluate the expression to a bool. If `false`, skip adding the job to configs but register skipped `once = true` jobs as exited. If `true`, emit the job with no `condition` field (it passed). This avoids any shell-based condition evaluation and keeps the two-worlds separation clean.
- `for_each` with arrays and ranges: expand at lower time into individual ProcessConfigs named `{job}-0`, `{job}-1`, etc. For `glob()`: store as `ForEachConfig` for runtime expansion.
